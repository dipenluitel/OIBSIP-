# Task 9 - Advanced SQL Injection Assessment

## Objective

This project demonstrates the analysis of SQL Injection vulnerabilities using Damn Vulnerable Web Application (DVWA) running locally.

Testing was performed only on a local training environment.

---

## Environment

- Windows 11
- XAMPP
- DVWA
- PHP 8.0
- MySQL
- Burp Suite Pro Edition
- WSL Ubuntu

---

## Security Level

Medium

---

## Methodology

1. Configure DVWA
2. Set Security Level to Medium
3. Open SQL Injection module
4. Observe application behaviour
5. Capture HTTP request using Burp Suite
6. Document findings
7. Explain mitigation techniques

---

## Burp Suite Analysis

Burp Suite Community Edition was used to intercept the HTTP request sent from the DVWA SQL Injection page while the application was running locally.

### Request Details

- Method: POST
- Target: `/dvwa/vulnerabilities/sqli/`
- Host: `192.168.1.68`
- Content-Type: `application/x-www-form-urlencoded`
- Security Level: Medium

### Request Body

```text
id=1&Submit=Submit
```
---

## Findings

The SQL Injection page accepted a user identifier and returned corresponding records.

Compared with the Low security setting used in Task 3, the Medium setting introduces additional input handling, making manual testing more restrictive.

Burp Suite was used to inspect the HTTP request generated by the application.

---

## Business Impact

SQL Injection vulnerabilities can expose confidential information stored in databases.

Potential impacts include:

- Data leakage
- Unauthorized database access
- Credential theft
- Financial loss
- Reputation damage

---

## Recommendations

- Prepared Statements
- Parameterized Queries
- Input Validation
- Least Privilege Database Accounts
- Web Application Firewall
